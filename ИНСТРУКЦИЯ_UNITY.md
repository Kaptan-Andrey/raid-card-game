# Что нужно сделать в Unity

Документ описывает, что делать после правок кода. Все скрипты — в `Assets/Scripts/`.

Разделы 1–4 — запуск (как раньше). **Главное новое — раздел 5: боевой UI.**
Там по шагам, какие объекты создать и какое поле куда привязать.

---

## 1. Открыть проект и дать ему скомпилироваться

1. Открой проект в Unity, открой сцену **`Assets/reid.unity`**.
2. Дождись пересборки скриптов.
3. Открой **Console** (Window → General → Console) — красных ошибок быть не должно.

> Если Mirror ругается на отсутствующие типы (`Batcher`, `Channels`, `Pool`…) —
> сначала почини Mirror: удали `Assets/Mirror`, сделай `git reset --hard origin/main`
> и `git clean -fd Assets/Mirror` (полная версия v96.10.0 уже в git).

---

## 2. Быстрый запуск (один игрок)

1. **Play** → в меню Mirror (NetworkManagerHUD) нажми **Host (Server + Client)**.
2. Раздадутся карты, появится Лес (4 карты), разложится зона «Рейд», сверху — фаза.
3. В фазе **Добор** кликай по Лесу/колоде, чтобы добрать.
4. Нижняя кнопка: в Доборе — завершает добор, в Действиях — заканчивает ход.

## 3. Тест на двух игроков

- **Вариант А:** Build сцены `reid.unity` → запусти `.exe`, нажми **Host**; в редакторе
  нажми **Play** → **Client** (адрес `localhost`).
- **Вариант Б:** ассет **ParrelSync** — два редактора без билда.

## 4. Переключатель `autoResolveTrophy` на GameManager

- **Включён (по умолчанию):** спелых пленных сервер забирает сам — игра не зависает.
- **Выключи**, только когда соберёшь **окно трофея** (раздел 5.3). Иначе игра встанет
  в фазе Трофеев.

# 5. БОЕВОЙ UI — какие объекты создать

Я доработал код так, чтобы:
- **карта показывала себя** (имя, силу, племя/тип, зёрна, метку защиты);
- была **панель отряда** (что выбрано, какое племя, суммарная сила);
- работал **режим усиления защиты**;
- было **окно выбора трофея** (Убить / Магия).

> **Все ссылки на эти объекты в инспекторе — необязательные.** Если что-то не привязать,
> игра не упадёт — просто эта часть UI не будет видна. Привязывай по мере готовности.

> **ВАЖНО про кнопки:** обработчики кликов навешиваются **из кода** (метод `WireBattleButtons`).
> Тебе НЕ нужно добавлять `OnClick ()` вручную в инспекторе кнопок — только перетащить
> сами кнопки в поля компонента `UIManager`.

Тексты — это компонент **TextMeshPro**. Создавать: правый клик в Hierarchy →
**UI → Text - TextMeshPro** (при первом разе Unity предложит «Import TMP Essentials» —
согласись).

---

## 5.1. Префаб карты — показать данные карты

Найди префаб, который привязан к `UIManager → Card Prefab` (обычно
`Assets/Prefabs/Card.prefab`). Открой его двойным кликом (Prefab Mode) и добавь
дочерние объекты. На корне префаба уже висит компонент **CardView** — туда и привязываем.

Рекомендуемая иерархия префаба:

```
Card                         (Image-подложка, CanvasGroup, CardView)  ← корень
├── Background   (Image)          → CardView.Background Image  (фон, красится по племени)
├── NameText     (Text TMP)       → CardView.Name Text
├── StrengthText (Text TMP, крупно, в углу)  → CardView.Strength Text
├── TribeType    (Text TMP, мелко)→ CardView.Tribe Type Text
├── GrainsText   (Text TMP)       → CardView.Grains Text     (значки ●○◐)
├── DefenderMark (Image, напр. щит)→ CardView.Defender Mark   (метка «в защите»)
└── SelectionBorder (Image, жёлтая рамка)→ CardView.Selection Border (рамка выбора)
```

Как привязать: выдели корень `Card`, в инспекторе у компонента **CardView**
перетащи каждый дочерний объект в одноимённое поле.

Поля **Face Root / Back Root** на префабе карты можно **оставить пустыми** — для
рубашки используется отдельный префаб `Card Back Prefab` (он уже настроен).

> Минимум для боя: **NameText**, **StrengthText**, **SelectionBorder**. Остальное —
> для красоты/удобства. `Background Image` красит карту по племени автоматически.

После правок Prefab Mode выйди (стрелка ← вверху сцены) — изменения применятся ко всем картам.

## 5.2. Панель отряда (нижняя полоса)

На главном **Canvas** сцены создай пустой объект-контейнер (можно с Image-подложкой),
назови `BattlePanel`, размести снизу. Внутри:

```
BattlePanel
├── SquadInfoText (Text TMP)  — две строки: режим + состав отряда
├── MessageText   (Text TMP)  — подсказки/ошибки (гаснут через 4 сек)
├── ReinforceButton (Button)  — внутри Text "Усилить защиту"
└── CancelButton    (Button)  — внутри Text "Сброс"
```

Привязка: выдели объект со скриптом **UIManager** (тот же, где уже заданы контейнеры
рук/полей). В инспекторе в разделе **«Боевой UI»**:

| Поле UIManager      | Что перетащить        |
|---------------------|-----------------------|
| `Squad Info Text`   | SquadInfoText         |
| `Message Text`      | MessageText           |
| `Reinforce Button`  | ReinforceButton       |
| `Cancel Button`     | CancelButton          |

Кнопки — просто **UI → Button - TextMeshPro**. OnClick не настраивай, код сам навесит.

---

## 5.3. Окно выбора трофея

На Canvas создай панель (Image, по центру), назови `TrophyPanel`. **Выключи её галочкой
активности** вверху инспектора (код всё равно скроет её на старте). Внутри:

```
TrophyPanel  (Image; объект выключен по умолчанию)
├── PromptText  (Text TMP)  — «Пленный … созрел. Забрать или магия?»
├── KillButton  (Button)    — внутри Text "Убить"
└── MagicButton (Button)    — внутри Text "Использовать магию"
```

Привязка в **UIManager → «Окно выбора трофея»**:

| Поле UIManager        | Что перетащить |
|-----------------------|----------------|
| `Trophy Panel`        | TrophyPanel    |
| `Trophy Prompt Text`  | PromptText     |
| `Trophy Kill Button`  | KillButton     |
| `Trophy Magic Button` | MagicButton    |

Когда окно собрано — можно **выключить** `Auto Resolve Trophy` на GameManager, и трофеи
будешь забирать вручную через это окно.

## 6. Как этим играть (фаза Действий)

1. **Выбор отряда** — кликай по картам в своей руке. Выбранная подсвечивается рамкой
   и слегка увеличивается. Внизу (`SquadInfoText`) видно: племя и суммарную силу.
2. **Атака** (режим по умолчанию) — кликни по цели:
   - карта в зоне **Рейд** → захват поселенца;
   - **пленный противника** → перехват.
   Сервер проверит: одно племя и сила **строго больше**. Если базовая сила отряда
   меньше — внизу появится предупреждение, но команда всё равно уйдёт (вдруг есть
   бонусы свойств). Удачная атака сама очистит выбор; неудачная — оставит отряд, чтобы
   добрать карты и повторить.
3. **Усиление защиты** — нажми **«Усилить защиту»** (режим переключится), выбери карты
   руки и кликни по **своему** пленному. Кнопкой **«Сброс»** — выйти из режима.
4. **Трофеи** — если `Auto Resolve Trophy` выключен, в фазе Трофеев откроется
   `TrophyPanel`: жми «Убить» (зёрна засчитаются к победе) или «Магия».

## 7. Что работает / чего ещё нет

**Работает (логика на сервере + UI):**
- Раздача, фазы, добор, накопление зёрен, **бой** (Рейд/перехват), **усиление**,
  автосбор/ручной сбор трофеев, **условие победы** (4 зерна любых ИЛИ 3 одного цвета).
- Эффекты зёрен (Светлое — добор, Тёмное — сброс защиты).

**UI пока НЕ сделан** (серверные команды есть — `player.CmdUseGrain`,
`player.CmdDiscardMagicForGrains`):
- Кнопки **использования зёрен** и **сброса магии ради зёрен** — это следующий шаг.
- **Зерно Покоя** (реактивное в ход противника) и индивидуальные **свойства карт**.

## 9. Если что-то не так — чек-лист

- **Ошибки компиляции** → пришли текст из Console.
- **Карты пустые (нет имени/силы)** → на префабе карты не привязаны `Name Text` /
  `Strength Text` в компоненте CardView (раздел 5.1).
- **Панель отряда/окно трофея не появляются** → не привязаны поля в UIManager
  (разделы 5.2 / 5.3). Это не ошибка — просто UI не подключён.
- **Карты не появляются вообще** → проверь контейнеры и `cardPrefab`/`cardBackPrefab`
  в UIManager.
- **Игра встала в фазе Трофеев** → включи `Auto Resolve Trophy` (пока нет окна) или
  собери окно из 5.3.
- **Второй игрок не появляется** → у NetworkManager `Player Prefab` задан и стоит
  `Auto Create Player`.
