# Что нужно сделать в Unity

Документ описывает, что делать после того, как я дописал серверную логику игры
(бой, трофеи, зёрна, победа). Все скрипты уже лежат в `Assets/Scripts/`.

> **Главная хорошая новость:** сцена `Assets/reid.unity` уже настроена.
> Мои изменения — это только код. **Никаких новых привязок в инспекторе делать НЕ нужно.**
> Можно просто открыть проект, дождаться компиляции и запускать.

---

## 1. Открыть проект и дать ему скомпилироваться

1. Открой проект в Unity.
2. Открой сцену **`Assets/reid.unity`** (не `Scenes/SampleScene.unity`).
3. Дождись, пока Unity пересоберёт скрипты (внизу справа крутится индикатор).
4. Открой окно **Console** (Window → General → Console) и убедись, что **нет красных ошибок компиляции**.

Если ошибок нет — всё на месте, переходи к запуску.

> Я менял структуру `CardData` (добавил поля) и её сетевой сериализатор
> `CardDataSerializer.cs`. Unity подхватит это автоматически при компиляции —
> вручную ничего регистрировать не надо.

---

## 2. Быстрый запуск (один игрок, проверка что игра живёт)

1. Нажми **Play**.
2. В левом верхнем углу появится меню Mirror (**NetworkManagerHUD**). Нажми **Host (Server + Client)**.
3. Должно произойти:
   - раздадутся карты в руку,
   - в «Лесу» появятся 4 карты,
   - в зоне «Рейд» разложатся поселенцы,
   - сверху покажется текущая фаза («Фаза: Draw» и т.д.).
4. В фазе **Draw** кликай по карте в Лесу или по колоде, чтобы добрать (до 5 карт, не больше 1 из Леса за ход).
5. Кнопка снизу:
   - в фазе **Draw** — завершает добор и переходит к Действиям,
   - в фазе **Action** — заканчивает ход.

---

## 3. Тест на двух игроков

Mirror — это клиент-сервер, поэтому нужны два «экземпляра» игры.

**Вариант А (проще):**
1. Сделай билд: File → Build Settings → добавь сцену `reid.unity` → Build.
2. Запусти собранный `.exe` — нажми в нём **Host**.
3. В редакторе Unity нажми **Play** → в HUD нажми **Client** (адрес `localhost`).
4. Теперь два игрока подключены, ходы чередуются.

**Вариант Б (удобнее для разработки):** поставь ассет **ParrelSync** —
он клонирует проект, и можно запускать два редактора (один Host, второй Client)
без билда.

> Текущий `AutoHost.cs` (авто-старт хоста) в сцене **не подключён** — стартуем вручную
> через HUD. Если хочешь авто-хост, повесь компонент `AutoHost` на любой объект сцены.

---

## 4. Новый переключатель на GameManager: `autoResolveTrophy`

Найди в сцене объект с компонентом **GameManager** → в инспекторе появится галка
**Auto Resolve Trophy** (по умолчанию **включена**).

- **Включена (true):** когда у пленного накопились зёрна до максимума, сервер
  **сам** забирает его в Трофей. Удобно, чтобы цикл игры и победа работали,
  пока нет окна выбора. **Оставь так для теста.**
- **Выключена (false):** сервер будет ждать решения игрока (Убить / Использовать магию).
  Это потребует UI-окна выбора, которого пока нет (см. раздел 6) — без него игра
  «зависнет» в фазе Трофеев. Выключай только когда сделаешь это окно.

## 5. Что уже РАБОТАЕТ (логика на сервере)

- Раздача колод, 62 монстра + 13 поселенцев, зона Рейда, Лес.
- Фазы хода: Обновление → Сбор трофеев → Добор → Действия.

## 6. Что пока НЕЛЬЗЯ сделать мышкой (нет UI) — но логика готова

Серверные команды написаны и вызываются, **но кнопок/окон для них в сцене ещё нет.**
То есть бой и зёрна сейчас можно запустить только из кода. Команды (методы на `Player`):

```csharp
// Атака: отряд карт (id), цель и её зона.
//  Рейд:     targetZone = ZoneType.Reid (или Reider), targetCardId = id поселенца,   targetPlayerIndex = -1
//  Перехват: targetZone = ZoneType.Player1Field/Player2Field, targetCardId = id пленного, targetPlayerIndex = индекс владельца
player.CmdAttack(int[] squadCardIds, ZoneType targetZone, int targetCardId, int targetPlayerIndex);

// Усиление защиты своего пленного картами с руки (все одного племени)
player.CmdReinforce(int prisonerId, int[] cardIds);

// Выкачать верхнее зерно пленного и применить эффект
//  targetPlayerIndex/targetPrisonerId — нужны только для Тёмного зерна
//  drawFromForest — только для Светлого (true = из Леса, false = из Колоды)
player.CmdUseGrain(int prisonerId, int targetPlayerIndex, int targetPrisonerId, bool drawFromForest);

// Сбросить карту "Магия" ради всех её зёрен
player.CmdDiscardMagicForGrains(int magicCardId, int targetPlayerIndex, int targetPrisonerId, bool drawFromForest);

// Выбор судьбы спелого пленного (когда autoResolveTrophy = false)
player.CmdResolveTrophy(int prisonerId, TrophyChoice.Kill);     // или TrophyChoice.UseMagic
```

**Чтобы дать это игроку мышкой — нужен боевой UI** (выбор отряда → клик по цели,
окно «Убить / Магия»). Это следующий большой шаг, который я могу сделать.

**Не реализовано совсем:** Зерно Покоя (реактивный эффект в ход противника —
требует системы «окон реакции») и индивидуальные Свойства карт.

## 7. ВАЖНО: заполнить реальные данные карт

Файл **`Assets/Scripts/CardDatabase.cs`** — единственное место, где заданы все карты.
Сейчас там **заглушки** (помечены `// TODO`), потому что этих чисел нет в буклете правил —
они напечатаны на самих картах. Нужно сфотографировать карты и вписать:

| Что | Где в файле |
|-----|-------------|
| Сила каждого монстра | `PLACEHOLDER_MONSTER_STRENGTH`, метод `Monster(...)` |
| Какие монстры — Афферные твари | поле `isAdferous` в `Monster(...)` |
| Какие монстры — тип «Магия» (без силы, со значками зёрен) | `cardType = CardType.Magic`, `magicGrains` |
| Сила поселенцев (цель для Рейда) | `PLACEHOLDER_SETTLER_STRENGTH` |
| Раскладка зёрен по делениям трекера у поселенцев | вызовы `Settler(id, "...", D, L, C, ...)` |

Пока числа — заглушки, играть можно, но баланс ненастоящий.

## 8. Если что-то пошло не так — чек-лист

- **Ошибки компиляции в Console** → пришли мне текст ошибки.
- **Карты не появляются / поле пустое** → проверь, что у объекта `UIManager` в сцене
  заполнены все поля-контейнеры (Player1HandContainer и т.д.) и `cardPrefab`/`cardBackPrefab`.
  *(Я проверил — сейчас всё заполнено, но если перетаскивал объекты, могло слететь.)*
- **Второй игрок не появляется** → у `NetworkManager` должно быть `Player Prefab = PlayerPrefab`
  и галка `Auto Create Player`. *(Сейчас стоит.)*
- **Игра встала в фазе Трофеев** → включи галку `Auto Resolve Trophy` на `GameManager`.
- **Перетаскивание карт не работает** → на зонах должен висеть `DropZone` с выбранным
  `Zone Type`, на префабе карты — `CardView` + `Canvas Group`. *(Сейчас всё на месте.)*

## Управление боем (фаза Действий) — кликами по картам

UI собран на кликах (отдельные кнопки не нужны):
1. **Выбор отряда** — кликай по картам в своей руке. Выбранная карта слегка
   увеличивается. Повторный клик — снять выбор.
2. **Атака** — кликни по цели:
   - карта в зоне **Рейд/Рейдер** → захват из Рейда;
   - **пленный противника** (его поле) → перехват.
   Сервер проверит: одно племя, сила строго больше. Если условие не выполнено —
   ничего не произойдёт (отряд останется выбранным).
3. Сила считается с учётом свойств (Обжора, Щенок, Мёртвый Ош, Фаун-защитник).

Пока **нет UI** для: усиления защиты, использования зёрен, сброса магии ради
зёрен и выбора трофея (Убить/Магия) — серверные команды есть, кнопки добавим
позже. Поэтому `autoResolveTrophy` держим **включённым**.
